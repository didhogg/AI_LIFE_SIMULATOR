# P0-5 拍板记录（以此为准）

> 本文档记录 P0-5（RNG 与统一检定）阶段的设计决策。后续实现以本文为准，不得与之矛盾。

## 1. 属性项口径（方案B 已定稿）

**结论**：属性项 = `(主属性 + Σ副属性_i × 权重_i) / 2`

- 由检定配方表在消费点（即检定时）合成，不预先展平存储。
- 配方结构：每个检定项有一个 `主属性` 字段和一个 `副属性: [{ 键, 权重, 停用?, 中性缺省? }]` 列表，以及 `拓扑` 和 `宿主类型` 字段（见第9节）。
- 整体属性项进指纹（通过 `检定配方表` 字段，含 `拓扑`/`宿主类型`/`停用` 配置）。

## 2. 结果档口径（余量制，已定稿）

**结论**：`M = 公式值 − u`，其中 `u ∈ [0, 99]` 为 `rngFor` / `rngForFate` 的整数输出。

| 档位 | 条件 |
|------|------|
| 大胜 | M ≥ 40 |
| 胜 | 15 ≤ M ≤ 39 |
| 惨胜 | 1 ≤ M ≤ 14 |
| 败 | −24 ≤ M ≤ 0 |
| 溃 | M ≤ −25 |

- 切分界是**数据**（`检定档切分表Schema`），住玩法预设层，可被预设覆盖，进指纹。
- 默认值：`{ 大胜下限: 40, 胜下限: 15, 惨胜下限: 1, 败下限: -24 }`。

## 3. 天命换骰（选项B，已定稿）

**结论**：`rngForFate(seed, tick, channel, fateRerollIndex = 0)`

- `fateRerollIndex` 仅在玩家使用**天命重掷券**时 +1。
- P0-5 仅保留参数位，不实装用券逻辑（P1 接线）。
- 天命通道永不混入普通重掷盐，防止玩家普通重掷影响天命结果。

## 4. 钳制表（双层结构，已定稿）

**结论**：双层结构 `{ 按重要等级, 按字段 }`

- **字段级优先，重要等级兜底**。
- `按重要等级`：对 `路人 / 次要 / 重要 / 核心` NPC 分别设 Δ 上限。
- `按字段`：对具体字段路径键（如 `"属性.智慧"` / `"技能.*"`）设 `{ 单次Δ上限, 最小值, 最大值 }`。
- P0-5 仅建占位结构并进指纹；闸逻辑（真正拦截变更）在 **P0-6** 实装。

## 5. 子种子合成（FNV-1a，已定稿）

**禁止** XOR 折叠（`seed XOR tick`、`tick XOR salt`）——会导致不同输入组合产生相同子种子。

**正确方式**：将 `(存档种子, 拍号, 通道名, rerollSalt)` 四分量以 null 字节分隔拼成字符串，对整体做 FNV-1a 32-bit 哈希：

```
input = `${seed}\x00${tick}\x00${channel}\x00${salt}`
subSeed = fnv1a32(input)
```

**盐源语义（对齐发现D / A1）**：`rngFor` 的 `rerollSalt` 参数 = **存档头·全局回滚计数器**。
- 每次玩家回滚/重掷时 +1，永不随快照还原，结构性堵死刷骰。
- P0-5：调用方直接传入。P0-9 将落地到 `$存档头.全局回滚计数器`。

**两者正交**：`$会话状态.演出层草稿计数`（原「本拍重掷序号」·发现D 改名）是纯叙事血统水印，绝不传入 `rngFor` 的 `rerollSalt` 参数，绝不进判定。

## 6. 指纹哈希取材范围（发现A 两分，已定稿）

`hashPresetFingerprint` 取材按**发现A 两分**，互斥不重叠，最终指纹 = 两部分经 `canonicalize()` 后合成一次 FNV-1a，输出 8 位十六进制：

### (a) 预设整包（`preset`，从不快照）
| 字段 | 说明 |
|------|------|
| `检定配方表` | 属性项方案B (主属性 + Σ副×权重)/2 |
| `检定档切分表` | 余量制 M 切分界（默认大胜40/胜15/惨胜1/败−24） |

### (b) 快照锁定（`snapshot`，开局锁定·随档快照）
| 字段 | 说明 |
|------|------|
| `难度系数组` | DC 偏置；指令组边界可切，每次切换更新指纹 |
| `判定骰型` | d100 / d20 |
| `暴击映射` | B1b：进指纹（判定口径，能把胜抬成大胜）；P0-5 预留字段位 |
| `钳制表` | P0-5 占位；P0-6 闸逻辑接线 |

调用方从**档内快照**传入 `snapshot` 组，绝不读 live 预设（防止快照回滚后指纹漂移）。

## 7. 实装范围（P0-5 边界）

P0-5 **仅做**：
1. Schema 增补（`$存档种子`、`$会话状态.演出层草稿计数`（原`本拍重掷序号`，发现D 改名）、`检定档切分表`、`钳制表`）。
2. `rngFor` / `rngForFate` / `hashPresetFingerprint` 纯函数（含发现A 两分重构）。
3. 检定函数返回原始 `u`（不含档判断逻辑）。

P0-5 **不做**：
- `runTick` 中的指纹写入（P0-7）。
- `$存档种子` 的开局生成与接线（P0-7）。
- `$存档头.全局回滚计数器` 落地（P0-9）。
- 钳制表闸逻辑（P0-6）。
- 天命重掷券用券逻辑（P1）。
- d20 吸附完整实装（P1）。

## 8. H2 数值安全：assertFinite NaN 源头盘点（P0-5 新增）

### 动机

`NaN > DC` 恒为 `false`；`NaN` 流入检定链会导致**静默失败**（tier 永远为最低档、世界卡住）而无报错。必须在边界处 fail-fast 拦截。

### NaN 常见来源清单

| 来源 | 示例 | 防御策略 |
|------|------|----------|
| `0 / 0` 除法 | 空分母统计 | 调用前断言分母 ≠ 0 |
| `Infinity − Infinity` | 出界值相减 | `assertFinite` 守卫输入 |
| `Math.log(负数)` | 对负余量取对数 | 使用 `fixedLog1p`；调用前值域检查 |
| 对 `undefined` 直接算术 | `属性轴数据[缺键] + n` | `?? 0` 后 `assertFinite`；`noUncheckedIndexedAccess` 辅助 |
| 空数组聚合（无初始值） | `[].reduce((s,x)=>s+x)` | 始终提供初始值 0 |
| 恶意 / 残损预设字段 | `权重: NaN` | `resolveAttribute` 入口 `assertFinite(权重, …)` |

### 双卡口设计

| 卡口 | 位置 | 状态 |
|------|------|------|
| 第一卡口 Gate ① | `check()` / `resolveAttribute()` 入口 + 每个公式产出 | ✅ P0-5 已实装 |
| 第二卡口 Gate ② | 余量M / 公式值写入涟漪/状态 delta 前（第⑤闸入账前） | ⚑ P0-6 占位；`TODO(P0-6)` 注释已在 `check.ts` 标注 |

### 实装规范

- **开发态**：`assertFinite` 直接 `throw FiniteAssertionError`，栈迹指向具体派生路径（`ctx` 字段）。
- **生产态**：`console.error` 记录后 `throw`，确保即使上层捕获也留有日志。
- **P0-7 消费**：`runTick` catch `FiniteAssertionError` → 回滚到拍前锚点（P0-7 实装）。
- **纪律**：新增数值派生路径必须在 Gate ① 前加 `assertFinite`；PR reviewer 检查项。

## 9. 检定配方表收尾：拓扑 + 宿主类型 + 停用轴（6.45 / 6.48）

### 9.1 检定拓扑（`拓扑: '即掷' | '骰池'`）

| 值 | 语义 | 状态 |
|----|------|------|
| `即掷` | 行动发生时当场掷骰（现状） | ✅ P0-5 实装 |
| `骰池` | 拍首预掷 N 骰入池，行动时消费 | ⚑ P0-2 占位；`resolveAttribute` 遇此值 throw 未实装 |

### 9.2 宿主类型（`宿主类型: '角色' | '组织' | '世界域'`）

| 值 | 数据来源 | 状态 |
|----|----------|------|
| `角色` | NPC 属性轴数据（`属性轴数据` 入参） | ✅ P0-5 实装 |
| `组织` | 组织属性轴（P0-1 schema） | ⚑ dispatch stub；throw 未实装 |
| `世界域` | 全局属性轴（P0-1 schema） | ⚑ dispatch stub；throw 未实装 |

### 9.3 停用轴中性缺省（`停用: boolean; 中性缺省: number`，6.48）

每条 `副属性列` 条目可声明 `停用: true` + `中性缺省: N`：

- 引擎取数时 **先判 `停用` 标志**，若为 `true` 则直接使用 `中性缺省`，完全不读轴数据。
- 防止"此角色类型不具备该轴"时因 `?? 0` 返回语义错误的 0，或在 NaN 场景污染结果。
- `中性缺省` 缺省为 0（`?? 0`）；调用方显式传入代表"此轴中立位"的数值（如 50）。
- `停用: false`（或缺省）时行为不变，仍读轴数据。

### 9.4 指纹覆盖

`拓扑`、`宿主类型`、`停用`、`中性缺省` 均嵌在 `检定配方表` 内，随 `preset.检定配方表` 整体进指纹（预设整包组）。AA6 gate D property 测试验证三个新字段变化均导致指纹改变。
