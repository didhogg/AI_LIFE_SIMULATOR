# P0-5 拍板记录（以此为准）

> 本文档记录 P0-5（RNG 与统一检定）阶段的设计决策。后续实现以本文为准，不得与之矛盾。

## 1. 属性项口径（方案B 已定稿）

**结论**：属性项 = `(主属性 + Σ副属性_i × 权重_i) / 2`

- 由检定配方表在消费点（即检定时）合成，不预先展平存储。
- 配方结构：每个检定项有一个 `主属性` 字段和一个 `副属性: [{ 键, 权重 }]` 列表。
- 整体属性项进指纹（通过 `检定配方表` 字段）。

## 2. 结果档口径（余量制，已定稿）

**结论**：`M = 公式值 − u`，其中 `u ∈ [0, 99]` 为 `rngFor` / `rngForFate` 的整数输出。

| 档位 | 条件 |
|------|------|
| 大胜 | M ≥ 40 |
| 胜 | 15 ≤ M ≤ 39 |
| 惨胜 | 1 ≤ M ≤ 14 |
| 败 | −24 ≤ M ≤ 0 |
| 溃 | M ≤ −25 |

- 切分界是**数据**（`检定档切分表Schema`），住玩法预设层，可被预设覆盖，进指纹。
- 默认值：`{ 大胜下限: 40, 胜下限: 15, 惨胜下限: 1, 败下限: -24 }`。

## 3. 天命换骰（选项B，已定稿）

**结论**：`rngForFate(seed, tick, channel, fateRerollIndex = 0)`

- `fateRerollIndex` 仅在玩家使用**天命重掷券**时 +1。
- P0-5 仅保留参数位，不实装用券逻辑（P1 接线）。
- 天命通道永不混入普通重掷盐，防止玩家普通重掷影响天命结果。

## 4. 钳制表（双层结构，已定稿）

**结论**：双层结构 `{ 按重要等级, 按字段 }`

- **字段级优先，重要等级兜底**。
- `按重要等级`：对 `路人 / 次要 / 重要 / 核心` NPC 分别设 Δ 上限。
- `按字段`：对具体字段路径键（如 `"属性.智慧"` / `"技能.*"`）设 `{ 单次Δ上限, 最小值, 最大值 }`。
- P0-5 仅建占位结构并进指纹；闸逻辑（真正拦截变更）在 **P0-6** 实装。

## 5. 子种子合成（FNV-1a，已定稿）

**禁止** XOR 折叠（`seed XOR tick`、`tick XOR salt`）——会导致不同输入组合产生相同子种子。

**正确方式**：将 `(存档种子, 拍号, 通道名, 重掷序号)` 四分量以 null 字节分隔拼成字符串，对整体做 FNV-1a 32-bit 哈希：

```
input = `${seed}\x00${tick}\x00${channel}\x00${salt}`
subSeed = fnv1a32(input)
```

## 6. 指纹哈希取材范围（已定稿）

`hashPresetFingerprint` 取材：`{ 检定配方表, 难度系数组, 钳制表, 判定骰型, 检定档切分表 }`，`JSON.stringify` 后 FNV-1a，输出 8 位十六进制。

## 7. 实装范围（P0-5 边界）

P0-5 **仅做**：
1. Schema 增补（`$存档种子`、`$会话状态.本拍重掷序号`、`检定档切分表`、`钳制表`）。
2. `rngFor` / `rngForFate` / `hashPresetFingerprint` 纯函数。
3. 检定函数返回原始 `u`（不含档判断逻辑）。

P0-5 **不做**：
- `runTick` 中的指纹写入（P0-7）。
- `$存档种子` 的开局生成与接线（P0-7）。
- 钳制表闸逻辑（P0-6）。
- 天命重掷券用券逻辑（P1）。
- d20 吸附完整实装（P1）。
