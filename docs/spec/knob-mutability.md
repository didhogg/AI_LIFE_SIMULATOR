## 相对早期草稿的修订（4 处）

- 🔴 **暴击映射**：原「不进指纹·P1 复审」→ **进指纹**（属「开局锁定·随档快照」组·B1b·判定口径，能把胜抬成大胜）。
- 🔴 **指纹取材**：原「五样揉一个哈希」→ **发现A 两分**（预设整包 / 快照锁定四样，互斥不重叠）。
- 🟡 **难度系数组 / 启用文风键**：原「拍边界可切」→ **指令组边界可切**（AA8 / 裁决六 / F1·分段点 = 组锚点）。
- 🟡 **盐 / 重掷序号**：盐源明确为 **存档头·全局回滚计数器**；「本拍重掷序号」→ **演出层草稿计数**（纯叙事水印·永不进盐 / 判定·发现D）。

## 核心原则

1. **开局锁定**：世界创建后不可改，只读进指纹。
2. **指令组边界可切**：可在指令组边界（组锚点）之间改，改后立即进指纹，**不下沉到内部拍**（AA8 / 裁决六）。
3. **随时可切**：随时可改，不进指纹、不影响历史回放。
4. **进指纹**：值变化导致 hashPresetFingerprint() 输出改变，触发 P0-7 重新快照。
5. **不进指纹**：不参与指纹哈希，改了不产生指纹差异。

## 指纹哈希取材范围（发现A 两分 · P0-5 定稿）

`hashPresetFingerprint()` 按发现A **两分、互斥不重叠**，最终指纹 = 两部分合成（各自 FNV-1a 32-bit，输出 8 位十六进制）：

- **预设整包**（从不快照的预设判定字段）：检定配方表、检定档切分表（及历法 / 粒度模板 / 母题配额 / 媒体渠道表等其它「从不变」判定面字段——本表只列检定相关）。
- **快照锁定**（开局锁定·随档快照·**从档内快照取、绝不读 live 预设**）：难度系数组、判定骰型、暴击映射、钳制表。
- **子种子 / 哈希纪律**：**禁 XOR 折叠**（seed XOR tick 等会撞种）；统一把 (存档种子, 拍号, 通道名, 盐) 四分量以 null 字节分隔拼串后做 FNV-1a。

## RNG 通道命名规范（P0-5 定稿）

| 前缀 | 适用场景 | 函数 |
| --- | --- | --- |
| 检定:xxx | 普通属性 / 技能检定 | rngFor |
| 触发:xxx | 事件触发 / 随机分支 | rngFor |
| 天命:xxx | 天命重掷券检定 | rngForFate |

rngFor 拒绝 `天命:` 前缀并 throw；rngForFate 只接受 `天命:` 前缀，否则 throw。

## 盐 / 演出层草稿计数（发现D / A1 / AA10）

- **全局回滚计数器（存档头）= RNG 盐源**：回滚 / 重掷 +1、永不随快照载入还原（整包载入旧快照也滚不回）→ 结构性堵死刷骰；普通检定通道混盐，天命通道不混。
- **演出层草稿计数（原「本拍重掷序号」·发现D 改名）= 纯叙事血统水印**：玩家重掷时 +1，**永不进盐、永不进判定**；与全局回滚计数器两层正交（AA10），互不回填。
- **天命换骰**：仅通过 rngForFate 的 fateRerollIndex 参数（仅天命重掷券触发）控制，与普通重掷盐隔离。
