# 旋钮可变性矩阵（Knob Mutability Matrix）

> 本文档定义了哪些预设字段可在何时修改、是否计入指纹、以及是否对 AI 可见。
> 指纹字段变化会产生新的 `_tick.难度系数组指纹`，迫使引擎重新计算基准；非指纹字段的变化是"软改"，不影响历史检定的重放一致性。

## 核心原则

1. **开局锁定**：世界创建后不可修改，只读进指纹。
2. **拍边界可切**：可在两拍之间修改，修改后立即进指纹，不可在拍中途修改。
3. **随时可切**：随时可改，不影响指纹，不影响历史回放。
4. **进指纹**：字段值的变化会导致 `hashPresetFingerprint()` 输出改变，触发 P0-7 的重新快照。
5. **不进指纹**：字段不参与指纹哈希，修改不产生指纹差异。

| 字段 | 所在层 | 可变时机 | 进指纹 | AI 可见 | 备注 |
|------|--------|----------|--------|---------|------|

| `钳制表`（规则补丁覆盖） | `规则补丁` | 拍边界可切 | ✅ | ❌ | 规则补丁可覆盖基础层；覆盖值同样进指纹 |

## 指纹哈希取材范围（P0-5 定稿）

`hashPresetFingerprint()` 接受以下字段组合并产生 8 位十六进制指纹：

```typescript
{
  检定配方表,
  难度系数组,
  钳制表,
  判定骰型,       // 来自 检定骰面.判定骰型
  检定档切分表,
}
```

哈希算法：FNV-1a 32-bit，输入为上述对象的 `JSON.stringify`。

## 通道命名规范（P0-5 定稿）

Ring 0 RNG 通道使用结构化前缀，引擎在运行时强制校验：

| 前缀 | 适用场景 | 函数 |
|------|----------|------|
| `检定:xxx` | 普通属性/技能检定 | `rngFor` |
| `触发:xxx` | 事件触发/随机分支 | `rngFor` |
| `天命:xxx` | 天命重掷券检定 | `rngForFate` |

`rngFor` 拒绝 `天命:` 前缀并 throw；`rngForFate` 只接受 `天命:` 前缀并 throw（违规）。

## 重掷序号语义（P0-5 定稿）

- `$会话状态.本拍重掷序号`：玩家点击"重掷这一拍"时递增，由引擎作为 `rerollSalt` 传入 `rngFor`。
- **不随拍前快照回滚还原**（blueprint 4.11③）：即使状态机回滚，序号继续单调递增，防止玩家通过回滚刷骰。
- 天命通道不混普通重掷盐；天命换骰通过 `fateRerollIndex` 参数（仅天命重掷券触发）控制。

## P0-7 验收清单（待实装）

以下不变量因 P0-4/P0-5 未实装 `runTick`/拍前快照，须在 P0-7 补写测试：

- [ ] **拍计数还原、重掷序号不还原**：执行拍前快照回滚后，`_tick.拍计数` 回到拍前值，但 `$会话状态.本拍重掷序号` 保持回滚前的较大值，不随快照还原。
- [ ] **指纹更新**：`runTick` 执行后，`_tick.难度系数组指纹` 等于当拍预设数值面的 `hashPresetFingerprint()` 输出。
- [ ] **$存档种子 接线**：Ring 0 在开局/读档时生成一次 `$存档种子`，之后只读；AI prompt 组装时必须剥离。
