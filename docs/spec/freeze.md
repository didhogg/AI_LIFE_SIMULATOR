## M·1 · 表达式 / 谓词 DSL v1.0(对应 F 区)

<aside>
💬

**大白话**:这是「游戏内算式」的语法手册——规定「分家产 1/3」「压上全部现金」这类算式只能怎么写、零头算给谁,免得 AI 乱写或算出不确定的结果。

</aside>

**版本号**:`DSL-grammar v1.0`(进指纹取材集,见 M·2;升版走 U3 版本分段)

**设计口径(回顾)**

- 纯计算无副作用:禁任意函数、循环、条件嵌套、随机(随机只走 rngFor,永不进表达式)。
- 全程精确有理数(分子/分母整型),**永不出现浮点**——`/` = 精确有理除、保留分数;只在最终入账那一刻取整(消灭浮点不确定性,接 H4/H5)。
- 取整方向恒不利于发起者:成本向上、收益向下、分配向下(少分不超分);「发起者」由提案单方向槽 Z2 确定(接 H7/H3,环路结构性无套利)。
- 求值时点 = 拍首快照(V6);同拍多指令按到达序。
- 路径只读真账;引不存在路径 = 解析失败走失败策略(不落账 + ⚠,M4 同族)。
- **版本演化律(S-1 拍板 ✅·向后兼容)**:文法只许加(新运算符/函数/字段)、永不重定义或删除既有语义(`/`/min/max/clamp 等既有语义永久冻结);新引擎(v2+)永远能读旧存档(v1)的表达式、求值逐位一致——存量未结算便条(遗嘱/租约/触发条件)由新版直接接管、绝不迁移重写(重写=改语义=违律庚);迁移重写仅作大版本破坏性变更逃生口(罕用非默认)。CI/fixture gate:全部 v1 fixture 表达式在当前文法下 parse 通过且求值逐位一致。边界:表达式引用被 schema 删除的路径=路径解析失败走失败策略(M4 同族·非文法问题);存档只升不降故只需「新读旧」、永不需「旧读新」。

**形式文法(EBNF)**

```ebnf

表达式   = 加减式 ;
加减式   = 乘除式 , { ("+" | "-") , 乘除式 } ;
乘除式   = 一元式 , { ("*" | "/") , 一元式 } ;   (* "/" = 精确有理除,保留分数 *)
```

**文法之外的静态约束(求值器实装时强制)**

1. 路径必须解析到五道闸 schema 派生白名单内的真相变量,否则解析失败。
2. 谓词引 extensional 落账集合法;引 intensional 谓词限深 1 层,超深 / 自环 / 互环 → 导入期拒收(Q6/K6)。

4. **一处消歧决定**:`/` 统一为精确有理除(不另设「整数除」运算符),「分数 1/3」与「整数除」由「全程有理精确 + 入账时按方向取整」统一实现——这条同时消除浮点不确定与两种除法的歧义。
