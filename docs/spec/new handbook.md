# 〇之二 · 【签】mod 防倒卖作者签名 · 设计与归位（新机制详规）

<aside>
🔏

**大白话**：给每个 mod 包盖一个「火漆封印」。作者用只有自己有的私钥，在「这个包的内容哈希」上盖章（签名）；公钥是「防伪对照卡」，随包带或登记到平台，谁都能拿来验。改了内容、改了作者名、去了水印——封印立刻对不上。

</aside>

## 三字段（落 mod 注册表 4.8 · 全部可空零迁移）

- `作者公钥?: string` —— 防伪对照卡；随包分发或登记到平台；**公开**，任何人可验
- `签名?: string` —— 作者用**私钥**对「该包内容哈希」盖的火漆封印
- `签名算法?: string`（可选，缺省＝平台标准 **Ed25519 · 已通过 ✅**）—— 留着以后换更强算法（**前向兼容·只增不改**，同 S-1 / 数学函数库版本演化律）

## 七条实装口径

- **验签挂点 ＝ 导入闸**（与 K6 pack_id 认证**同一道安检**）：用 `作者公钥?` 验 `签名?` 是否盖在内容哈希上；验不过 → 按平台/预设策略**拒收**或打「未验证/疑似篡改」警示（并入既有警示族）
- **内容哈希复用 B1c**：被签的「内容哈希」＝「生效中内容包集哈希」里**该包那一份单包内容哈希**——零新哈希通道
- **不进指纹**：签名三字段是真实性/血统元数据、不影响判定算法 → 入**指纹排除名单**（与 lineage / 审计血统 / `来源选择器?` 同性质）；但被签的内容哈希本身已在指纹（B1c），**改内容→哈希变→签名失效**，双保险
- **确定性**：验签在**导入期（档外、不进 Ring 0 结算）**，给定(公钥,签名,消息)结果确定——不碰确定性六禁；验签库版本若影响结果，比照数学函数库版本演化律（只增不改、版本可入排除名单备查）
- **键名冻结**：`作者公钥?`/`pack_id` 一旦实例化进存档即冻结（K2/K6 同款），改名走 `显示名?`，不可改 schema 键
- **🟢 诚实边界（防当成万能 DRM）**：签名能证明「此包确由该作者发布 + 内容未被篡改」，防的是**去水印/改作者名/篡改内容后再分发**；**防不了「原样整包转发」**（封印仍有效）——真·防倒卖需平台侧绑定授权/许可证，引擎侧只负责「身份 + 完整性」两件，平台倒卖检测 ＝ 公钥注册库比对。这条必须写进 docs，防实装者把签名当 DRM。
- **与原生卡包通道（⑮）咬合**：原生包跳 L1 但不跳导入闸——验签在导入闸，原生/升格两条路都过验签，零豁免

# 〇之三 · X 批 媒介渲染与多源消息流（X1–X15 · 已终审入蓝图 6.70）

<aside>
📰

**大白话**：游戏里的报纸/日记/短信/论坛怎么生成、怎么存、怎么撤回。三句话——**事实即时记账、散文惰性生成、刷一次出一条**。

</aside>

**三口径**：①**Lazy＋Freeze 渲染律**——媒介成品按需首次生成 → 可 swipe 仅换措辞（事实已编年史落账不可重抽·与 6.69 律一同侧）→ 保存即固化进渲染缓存从此不可变；LLM 同 prompt 不可复现 ⇒ 必须缓存（E2 重生成兜底 = 事实/知情面不漂、措辞可漂）。②**单源/多源两分律**——单源整篇（报纸/告示/日记/信件）= 编年史媒介附件单 blob·文风随格式模板键；多源逐条（短信/群聊/论坛/弹幕）= 叙事流条目序列·每条带说话者实体键＋渠道分组·文风随发信实体派生。③**撤回＝追加事件律**——原消息永不删，撤回 = 追加修正事件指原条目；配 Lazy＋Freeze「来得及看到才知道内容」自动正确。

**三调度（媒体节律·十六字「通信落账、期刊函数、流即采样、积压折叠」）**：指向性通信（短信/来信/@你）= 真事件逐条落账；周期刊物（报纸日更/杂志月更）= 纯函数容器〔第 X 期 =(查询日−创刊日)÷周期·零存储·跳百年不生成一期〕；无限流（手机首页/热搜）= 按需采样视图〔feed(拍号,刷新序号)·seed 确定性〕；频道集挂纪元包（6.62）随生效锚点切换。

**X1–X15 任务按落点分散到下方各阶段**：X1/X2/X3/X4/X5/X15→阶段 1（P0-1c schema 字段族）；X6/X7/X8/X13→P0-8；X9/X11/X12→P0-7；X10/X14→P0-9；前端（回想 UI/收件箱/报纸合订本/GAL 框/多源短信线程）→P1。

# 〇之四 · 【调】LLM 调用参数 · 切片预算 · 媒体输入 · 设计与归位（新批 · 本轮加入）

<aside>
🎛️

**大白话**：玩家在设置面板里能拧的那排「模型旋钮」——温度、Top P、频率/存在惩罚、一次最大回复长度、流式、思维链、上下文 token 上限、发图——它们的字段住哪、谁拿去调 API、排哪个阶段。一句话定性：**core 不调 LLM，这些只是注册表/模型画像/预算控制台里的「数据」；宿主 adapter 拿去调 API；玩家面板 P1 才接上。**

</aside>

## 一 · 总定性（三条铁律）

- **数据 / 执行两分**：参数值是 core 里的数据（调用类型注册表 4.8 + `$模型画像` 4.9 + `$预算控制台`）；真正传给 API 在宿主 adapter（AA2 已定「preset 只承载模型/采样参数，主叙事正文走引擎组装器」）。
- **全部不进指纹**：采样参数 / 切片预算 / 最大长度 / 流式 / 思维链只改「AI 那张嘴」，不进判定算法 → 入**指纹排除名单**（与【签】三字段、`来源选择器?` 同性质）。拧它们不破坏确定性重放。
- **可空零迁移**：所有新字段在 P0-1 黄金窗口顺手预埋，缺省 = 平台出厂值；P0-6 焊死前免费加。

## 五 · 三样最难的解决方案与大框架

### ① 一次最大回复 token 长度

- **难点**：不是"砍字数"，是"AI 说到一半被截断"会产出半截结构化记账提案 / 半截叙事，破坏输出契约。
- **方案（双层）**：(a) 软层——输出契约里声明期望长度，prompt 给"控制在 N 字"软约束；(b) 硬层——API `max_tokens` 兜底，配 `finish_reason=length` 检测 → 结构化调用判废重试、叙事调用走"续写拼接"。
- **大框架**：调用类型注册表按"输出形态"分流（**结构化短** = 记账/校准，给小上限+截断即判废重试；**叙事长** = 故事，给大上限+截断续写）。截断处理策略写进注册表，不靠 AI 自觉。

### ② 上下文 token 上限（切片预算 · 6.64）

- **难点**：决定"往上下文里装什么"会撞两条红线——**知情过滤**（不能为省 token 泄露真值）和**可复现组装**（切片不进指纹但必须确定性装配）。
- **方案**：组装器里加"预算管家"，按 B1–B6（6.64 已定）的**四级供给阶梯 + 让位序**装配：①不可裁三件（当前事件事实 + 主角现状 + 本次 choices）②在场 NPC 认知投影 ③地点/战争/秘密（过知情过滤）④历史/风味。宿主侧 tokenizer 边填边估，逼近硬上限就从最低优先级往下砍，砍掉的记入"本次省略"供调试。
- **大框架**：切片装配 = **优先级队列 + token 预算约束**；预算值来自注册表默认 / 玩家覆盖层；**知情过滤是硬闸、永远先于预算执行**（绝不为塞进预算而绕过认知投影）；切片本身不进指纹，只缓存供重生成对账（"事实不漂、措辞可漂"）。

### ③ 发送内联媒体（多模态输入）

- **难点**：不是拧数字，是**新建一条多模态输入管线**+ 安全；且要与 6.70 媒介渲染分清——**6.70 是"游戏内生成报纸/短信"的输出，这里是"给 LLM 喂图/文件"的输入**，两码事。
- **方案（三件）**：(a) **能力声明**——`$模型画像.多模态能力?` 决定该 provider/模型能否收媒体，不能则回退链降级为"文字描述替代"；(b) **组装器扩展**——上下文组装器支持把媒体引用按 provider 格式编码进 prompt（base64 / url / file API），媒体作为"切片的一种条目类型"（文本切片 + 媒体切片）；(c) **安全**——媒体经净化（复用 IM4 HTML 净化同款思路）、体积走切片预算的媒体上限、不可信来源降级。

# 大白话词表

- **火漆封印（签名）**：作者用只有自己有的私钥，在「包的内容哈希」上盖章；公钥是防伪对照卡，谁都能验。改内容/改作者名/去水印 → 封印对不上。**但原样整包转发它拦不住**，那要靠平台授权。
- **黄金窗口**：P0-6 把「权限白名单自动派生」焊死之前，加 schema 字段免费；焊死后加要付迁移+重录全部基准。所以 schema 级的事现在不做最亏。
- **六禁 CI**：六条自动纠察，禁止「看墙上的钟、用因电脑而异的数学、按地区习惯排序」等会让两台电脑跑出不同结果的写法。
- **canonical 序列化**：算指纹前先把数据按固定顺序摆好再哈希，否则同内容因摆放顺序不同 → 不同指纹 → 假警报。
- **派生器 dry-run**：让程序「从数据结构自己长出权限白名单」，先空跑确认每个字段权限层级分对、每个动词落账点都被覆盖——现在缺字段免费补。
- **指纹粒度（发现A）**：开局锁定的规则要拍「当时锁定的那张快照」的照片，不能拍「现在的预设文件」——否则中途被改了就对不上。
- **重放器**：从存档起点把每步输入重喂纯函数跑一遍、比对指纹，证明历史可复现没被偷改；盐值要读当时记录的值。
- **混沌猴**：自动跑一万拍并随机打断/乱序/并发，专抓想破头也想不到的时序 bug。
